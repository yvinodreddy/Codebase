**Document Version:** 1.0
**Date:** October 23, 2025
**Philosophy:** Pay competitively, reward excellence, ensure fairness

---

## EXECUTIVE SUMMARY

This framework establishes a transparent, performance-based compensation system that:

- **Attracts Talent:** Competitive stipends for skilled engineers
- **Rewards Performance:** Bonuses for exceptional work and milestones
- **Ensures Fairness:** Clear criteria, no favoritism
- **Motivates Team:** Story points, quality, collaboration, research
- **Retains Top Performers:** Performance multipliers and growth path

**Total Monthly Budget:** ₹8-10 lakhs for 22 resources
**Annual Budget:** ₹96 lakhs - ₹1.2 crores

## PERFORMANCE MULTIPLIERS

### Performance Rating System:

**5-Point Scale:**
- **1.0 - Underperforming:** Below expectations (80% of base)
- **1.1 - Meeting Expectations:** Solid contributor (100% of base)
- **1.2 - Exceeding Expectations:** Strong performer (110% of base)
- **1.3 - Outstanding:** Top 20% performer (120% of base)
- **1.4 - Exceptional:** Top 5% performer (140% of base)

### Calculation Example:

```
Base Stipend: ₹50,000
Performance Rating: 1.2 (Exceeding Expectations)

Monthly Stipend = ₹50,000 × 1.2 = ₹60,000
```

## PERFORMANCE EVALUATION CRITERIA

### Quantitative Metrics (50% of evaluation):

**For Developers:**

1. **Story Points Completed (20%):**
   - Target: Individual capacity based on role
   - Calculation: (Actual points / Target points) × 20%

2. **Code Quality (15%):**
   - SonarQube score: Target >80
   - Code review feedback: Average rating >4/5
   - Technical debt created: Minimal

3. **Bug Density (10%):**
   - Bugs per 1000 lines of code
   - Target: <1 critical/high bug per 1000 LOC

4. **Sprint Commitment (5%):**
   - % of committed stories completed
   - Target: >90%

**For Research Engineers:**

1. **Development Work (20%):**
   - Same as developers (50% capacity)

2. **Research Output (15%):**
   - Paper drafts submitted
   - Experiments completed
   - Research contribution quality

3. **Publication Progress (10%):**
   - Co-author on target papers
   - Review feedback incorporated

4. **Collaboration (5%):**
   - Working with other researchers
   - Academic connections made

**For Leadership:**

1. **Team Velocity (20%):**
   - Team's average velocity vs. target

2. **Sprint Completion Rate (15%):**
   - % of sprints meeting goals

3. **Quality Metrics (10%):**
   - Team's code quality average
   - Test coverage

4. **Team Satisfaction (5%):**
   - Anonymous team survey results

---

### Qualitative Metrics (50% of evaluation):

**All Roles:**

1. **Collaboration (15%):**
   - Helping team members
   - Code review quality
   - Knowledge sharing
   - Team player attitude

2. **Communication (10%):**
   - Clear updates in standups
   - Documentation quality
   - Responsiveness
   - Transparency

3. **Problem-Solving (10%):**
   - Proactive issue identification
   - Creative solutions
   - Handling blockers
   - Learning from failures

4. **Initiative (10%):**
   - Going beyond assigned tasks
   - Suggesting improvements
   - Self-learning
   - Ownership mentality

5. **Culture Contribution (5%):**
   - Positive attitude
   - Supporting team morale
   - Mentoring others
   - Embodying values

## PERFORMANCE REVIEW PROCESS

### Monthly Performance Review:

**Week 1 of Each Month:**

**Step 1: Self-Assessment (30 mins)**
- Team member completes self-assessment form
- Rates own performance on quantitative and qualitative metrics
- Lists accomplishments and challenges

**Step 2: Tech Lead Review (60 mins total, 1-on-1)**
- Tech Lead reviews metrics and self-assessment
- Provides feedback on performance
- Discusses growth areas
- Sets goals for next month
- Assigns performance rating (1.0-1.4)

**Step 3: Stipend Calculation**
- Tech Lead calculates stipend with multiplier
- Submits to Project Director for approval
- Payment processed

**Step 4: Feedback Documentation**
- Performance review notes documented
- Shared with team member
- Stored for future reference

---

### Quarterly Performance Deep-Dive:

**Every 3 Months:**

**1-on-1 with Project Director (45 mins)**
- Review quarterly performance trends
- Discuss career development
- Identify training needs
- Adjust compensation tier if warranted (promotion)
- Set quarterly goals

**Potential Outcomes:**
- **Promotion:** Move up a tier (e.g., Mid-level to Senior)
  - Requires: Consistent 1.3+ ratings, demonstrating senior-level work
  - Increase: ₹5,000-₹10,000 base increase
- **Pip (Performance Improvement Plan):** If consistently underperforming
  - 30-day plan with clear goals
  - Weekly check-ins
  - Support and resources provided
- **Continuation:** Stay in current tier with feedback

---

## PERFORMANCE IMPROVEMENT PLAN (PIP)

**Trigger:** 2 consecutive months with 1.0 rating (Underperforming)

**Process:**

**Week 1: PIP Initiation**
- Meeting with Tech Lead and Project Director
- Specific performance issues identified
- Clear goals set (must be achievable)
- Support offered (training, mentoring, etc.)
- 30-day timeline set

**Weeks 2-4: Weekly Check-ins**
- Progress review every Friday
- Feedback and course correction
- Documentation of improvement

**Week 5: PIP Review**
- Final assessment
- **Outcomes:**
  - **Success:** Return to normal performance reviews, stipend restored to 1.1× base
  - **Partial Success:** Extend PIP by 15 days with adjusted goals
  - **Failure:** Mutual separation with 15-day notice + severance (1 month stipend)

**Rationale:** Give team members fair chance to improve while maintaining project quality

## COMPENSATION PHILOSOPHY

### Principles:

1. **Transparency:** Everyone knows how stipends are calculated
2. **Meritocracy:** Performance determines pay, not favoritism
3. **Fairness:** Equal pay for equal work and performance
4. **Growth:** Clear path to higher compensation through performance
5. **Team Success:** Bonuses reward team achievements, not just individual
6. **Balance:** Mix of guaranteed (base) and variable (performance) compensation

### Pay Equity:

- **Same role, same tier, same performance = same pay**
- No discrimination based on gender, background, location, etc.
- Only factors: role, experience level, performance
- Annual pay equity audit
