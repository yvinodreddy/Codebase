ClaudePrompt is a Python-based orchestration framework built on Claude Code (Anthropic's Claude Sonnet 4.5 model) that implements a six-stage processing pipeline to ensure production-grade AI outputs. The system architecture follows microservices design principles with clear separation of concerns across 31 component files organized into four primary modules: Agent Framework (12 files), Guardrails (7 files), Security (7 files), and Core System (5 files).

**Six-Stage Processing Pipeline**

1. **Stage 1 - Prompt Preprocessing & Analysis**: Analyzes incoming prompts for length (up to 200,000 tokens), complexity (categorized as SIMPLE, MODERATE, or COMPLEX), and resource requirements. Dynamic agent allocation ranges from 5 to 500 agents based on complexity assessment.

4. **Stage 4 - Agent Orchestration**: Dynamically allocates and coordinates 5-500 parallel agents with priority-based scheduling (CRITICAL for guardrails, HIGH for core functionality, MEDIUM for parallel workload distribution). Agent types include Input Analyzers, Context Gatherers, Security Validators, Task Executors, Verifiers, and Quality Assurance agents.
