================================================================================
COMMON MISTAKES TO AVOID
================================================================================

MISTAKE #1: Vague Description
------------------------------
‚ùå WRONG: description: "Helps with insurance stuff"
‚úÖ RIGHT: description: "Analyzes and compares health insurance plans
          (individual vs group), calculates costs, verifies networks, and
          generates formatted comparison reports for decision-making"

WHY IT MATTERS: Vague = skill won't activate when needed


MISTAKE #2: Not Testing Calculations
-------------------------------------
‚ùå WRONG: Write formulas, assume they're correct, upload immediately
‚úÖ RIGHT: Run cost_calculator.py, verify against manual calculations,
          test edge cases, THEN upload

WHY IT MATTERS: Wrong calculations = wrong recommendations = bad decisions


MISTAKE #3: Ignoring README.md
-------------------------------
‚ùå WRONG: "Claude doesn't read README, so I'll skip it"
‚úÖ RIGHT: Write comprehensive README for humans (future you, team members)

WHY IT MATTERS: 6 months later, you'll forget how it works. README saves you.


MISTAKE #4: Not Versioning Changes
-----------------------------------
‚ùå WRONG: Make changes, save over old file, upload
‚úÖ RIGHT: Update version number (1.0.0 ‚Üí 1.1.0), document what changed

WHY IT MATTERS: Can't rollback if new version has bugs


MISTAKE #5: Creating Skills for One-Time Tasks
-----------------------------------------------
‚ùå WRONG: "I need to analyze this one insurance quote, let me build a skill"
‚úÖ RIGHT: "I analyze insurance 4 times per year, skill will pay off"

WHY IT MATTERS: Skills have creation cost. Need 3+ uses to justify.


MISTAKE #6: Uploading Without Testing Components
-------------------------------------------------
‚ùå WRONG: Write all files ‚Üí ZIP ‚Üí Upload ‚Üí Hope it works
‚úÖ RIGHT: Test calculator ‚Üí Test template ‚Üí Test together ‚Üí THEN upload

WHY IT MATTERS: Debugging after upload is harder than debugging locally


MISTAKE #7: Not Checking Plan Level
------------------------------------
‚ùå WRONG: Spend hours creating skill ‚Üí Try to upload ‚Üí "Skills not available"
‚úÖ RIGHT: Check plan FIRST (need Pro/Max/Team/Enterprise)

WHY IT MATTERS: Can't use skills on Free plan (but components still useful!)
